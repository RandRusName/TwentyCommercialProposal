# Document Service Runbook

## Local Run

From `mikoton-commercial-proposals/`:

```powershell
$env:PYTHONPATH = (Resolve-Path .\document-service).Path
$env:DOCUMENT_SERVICE_SECRET = "<local-secret>"
& "C:\Users\user\.cache\codex-runtimes\codex-primary-runtime\dependencies\python\python.exe" `
  -m mikoton_document_service.server
```

Default address:

```text
http://127.0.0.1:8010
```

Checks:

```powershell
curl http://127.0.0.1:8010/healthz
curl http://127.0.0.1:8010/readyz
```

## Security

- Do not expose the service publicly.
- Set `DOCUMENT_SERVICE_SECRET`.
- Twenty App sends `Authorization: Bearer <DOCUMENT_SERVICE_SECRET>`.
- Do not log the secret or request Authorization header.
- Do not accept template paths from clients.

## Storage

Current MVP stores files locally under:

```text
generated-documents/
```

Returned URLs are local `file://` URLs. This is suitable for local generation
tests, but target deployment still needs production storage such as S3/MinIO or
Twenty Files integration before client delivery.

## PDF Engine

Current MVP PDF is generated by ReportLab from the normalized payload. It proves
the contract and file pipeline, but it is not yet an Excel print-area export.

Excel/LibreOffice based PDF export remains required before Phase 4 can be
considered production-ready.
